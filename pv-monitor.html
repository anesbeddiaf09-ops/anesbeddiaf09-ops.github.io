01 — Project Overview

What this system does

An end-to-end solution for monitoring solar panel performance, designed to be low-cost, portable, and easy to deploy in educational or small-scale installations.

Data Acquisition
The Arduino Mega collects voltage, current, power output, ambient temperature, humidity and light intensity simultaneously using four dedicated sensors.
Wireless Transmission
Measured data is streamed wirelessly via HC-06 Bluetooth module to a paired Android smartphone, enabling cable-free monitoring in real conditions.
Android Visualization
A custom app built with MIT App Inventor receives and displays all parameters in real time, offering an intuitive interface for performance analysis.
PV Performance
The combined data reveals how environmental conditions — sunlight, temperature, humidity — directly affect the photovoltaic panel efficiency at any moment.

MICROCONTROLLER

Arduino Mega 2560

The brain of the system. An advanced platform built around the ATmega2560 chip, chosen for its expansive I/O capacity and built-in serial communication ports.

  • 54 digital I/O pins (vs 14 on the Uno)
  • 16 analog input channels
  • Multiple hardware serial ports (UART)
  • Dimensions: 101.6 × 53.3 mm
  • Open-source, large community support

MOBILE INTERFACE

Android Smartphone

Acts as the human-machine interface. Receives Bluetooth data and renders it through a purpose-built Android application created in MIT App Inventor.

  • Real-time parameter visualization
  • Built-in Bluetooth receiver
  • Displays: voltage, current, power, temperature, humidity, light intensity
  • No internet required — direct local link
03 — Sensor Suite

Four sensors, complete picture

Each sensor handles a specific physical parameter, converting it into a signal the Arduino can process and transmit.

DHT22
Temperature & Humidity
ENV

A digital environmental sensor integrating a capacitive humidity element and an NTC thermistor. Data is transmitted serially every 2 seconds, requiring a 4.7–10 kΩ pull-up resistor on the DATA line.

  • Supply voltage3 – 5 V
  • Consumption2.5 mA
  • Temp. range−40°C to +80°C
  • Humidity range0% – 100% RH
  • Dimensions27×59×13.5 mm
  • Interface1-Wire Digital
Digital I/O pin — Arduino Mega
BH1750
Ambient Light Intensity
LIGHT

A high-resolution digital light sensor using the I²C protocol. Measures illuminance from 1 to 65,535 lux with internal analog-to-digital conversion — no external ADC needed. Critical for correlating solar irradiance with panel output.

  • Supply voltage4.5 V
  • Consumption2.5 mA
  • Measurement range1 – 65,535 lux
  • Operating current7 mA
  • Power260 mW
  • InterfaceI²C (SCL / SDA)
I²C Bus — SDA / SCL pins
ACS712
DC/AC Current Sensing
CURRENT

Based on the Hall effect principle. Connected in series with the load, it converts the conductor's magnetic field into an analog voltage proportional to current, at 100 mV/A sensitivity. Requires zero-current calibration before use to eliminate offset voltage.

  • Model selectedACS712-05B-T
  • Sensitivity100 mV / A
  • Output typeAnalog voltage
  • SupportsAC & DC circuits
  • CalibrationZero-current offset required
Analog pin A10 — Arduino Mega
B25 Sensor
DC Voltage Detection
VOLTAGE

A resistive voltage divider that scales down the panel voltage by a factor of 5, bringing it within the Arduino's 5V analog input range. Safely measures photovoltaic output up to 25V DC with fine resolution.

  • Input range0 – 25 V DC
  • Measurement start0.02445 V
  • Resolution0.00489 V
  • Scale factor÷ 5 (resistive divider)
  • Compatible with5V & 3.3V systems

04 — Communication

Wireless with HC-06

The HC-06 Bluetooth module bridges the Arduino and the Android phone, enabling real-time serial data streaming with zero infrastructure dependency.

Key Design Consideration

The HC-06 RX pin tolerates a maximum of 3.3V, but Arduino outputs 5V logic. A voltage divider is mandatory to protect the module:

// Voltage divider on RX line
Arduino TX (5V) → R1 (1kΩ) ──┬── HC-06 RX
                     └── R2 (2kΩ) → GND

Vout = 5V × 2/(1+2) = 3.33V

Operating voltage: 3.3 – 5V | Baud rate: 9600 bps (default)

05 — Implementation

From design to reality

A structured workflow ensured each phase — simulation, assembly, and validation — was completed with technical precision.

1
Virtual Schematic Design (Tinkercad CAD)
Before any physical wiring, the complete circuit was designed and validated on Tinkercad, a dynamic online CAD platform. This allowed testing of the electronic schematic and Arduino code logic in a safe simulation environment, preventing hardware damage.
2
Component Selection & Rigorous Wiring
Each sensor was selected based on its measurement range, accuracy, and compatibility with the Arduino's analog/digital inputs. Wiring followed precise pin assignments: DHT22 on digital I/O, BH1750 on I²C, ACS712 on A10, B25 on pin 9, HC-06 with voltage divider protection.
3
Arduino Firmware Programming
The Arduino sketch handles multi-sensor polling, signal conditioning (ACS712 zero-offset calibration, B25 scaling by factor 5), data formatting, and serial transmission over Bluetooth at 9600 baud. All readings are packaged into a structured string sent to the HC-06 module.
4
Android App Development (MIT App Inventor)
The Android application was developed using MIT App Inventor, a visual block-based environment. It handles Bluetooth device discovery and pairing, continuous data reception, parsing of the incoming serial string, and real-time display of all six parameters on screen.
5
Hardware Assembly & Experimental Validation
The physical system was assembled on a breadboard, connecting all sensors and the HC-06 module to the Arduino Mega. The full setup was tested under real photovoltaic panel conditions, validating data accuracy, wireless stability, and app responsiveness.
